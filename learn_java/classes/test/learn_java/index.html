Threads

Here we will write a simple model of a bank account. First we will get the bank account working without multithreading. Then we will create some threads that access the accounts. Then the fun part begins! Once we start running many threads simultaneously we will encounter some problems, which we will (hopefully) solve. These solutions may lead to more problems! Threads are fun!

The Depositor Thread

The Depositor is an object that runs in a thread (in fact, it will probably extend java.lang.Thread) and continuously deposits a certain amount into a certain account. In its constuctor it takes three parameters: the account to deposit into, the amount to deposit, and the number of times ro deposit before stopping. In the code: 

     1:   Thread depositor = new Depositor(account, 5, 1000);
     2:   depositor.start();
     3:   depositor.join();
Line 1 creates the depositor, intending to deposit $5, 1000x. Line 2 starts the thread. Line 3 (calling "Thread.join()") makes the test code thread wait until the depositor thread has completed.
